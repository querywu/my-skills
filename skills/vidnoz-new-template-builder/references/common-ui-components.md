# 公用 UI 组件参考

这份文件用于沉淀 `templates/new-template/Dev/js/` 中可跨页面复用的 UI 组件或交互模式。

重点记录：

- 组件用途
- 典型参数
- 典型触发方式
- 适用于普通页面还是 Gen 页面
- 当前是否已有共享实现

这份文件当前沉淀的下载与错误弹窗模式，都只作为普通 / 非 Gen 页面参考。

Gen 页面有自己的下载与错误处理方法，分析或实现 Gen 页面时，不要默认复用这里的模式。

## 非 Gen 下载组件

来源样本：

- `templates/new-template/Dev/js/ai-video-compressor.js`

### 当前实现方式

这个“下载组件”在当前样本里不是单独命名的组件函数，而是通过 `ToolTip(...)` 的 `progress` 模式实现下载过程弹层。

### 当前用途

- 展示下载进度
- 处理下载成功
- 处理下载失败
- 处理资源不存在

### 当前参数结构

样本中使用了：

- `type: "progress"`
- `content`
- `progressType: "fetch"`
- `url`
- `name`
- `sucessCallback`
- `failedCallback`
- `NoExistCallback`

### 当前调用前置链路

在 `ai-video-compressor.js` 里，下载流程是：

1. 先根据资源 key 请求访问地址
2. 成功后拿到 `downloadUrl`
3. 再调用 `ToolTip({ type: "progress", ... })`
4. 由回调分别处理成功、失败和资源不存在

### 使用判断

- 这是一个“非 Gen 下载弹层模式”
- 它不是单纯的错误提示，而是承担下载流程 UI
- 当普通页面存在下载进度需求时，应先搜索是否已经沿用这套 `ToolTip + progressType: "fetch"` 模式
- 如果当前页面属于 Gen 流程，默认不要直接复用这套下载弹层模式，应先检查该 Gen 页面已有的方法

## 非 Gen 错误弹窗

来源样本：

- `templates/new-template/Dev/js/ai-video-compressor.js`
- 仓库内多个普通页面都出现 `ToolTip(...)` 调用

### 当前组件名

- `ToolTip(...)`

### 当前用途

- 展示错误提示
- 展示成功提示
- 在部分页面中兼做下载进度提示

### 当前样本中的错误提示参数

常见参数包括：

- `type: "error"`
- `title`
- `content`
- `btn`

成功提示样本：

- `type: "success"`
- `title`
- `content`
- `btn`

### 当前使用特点

- 在仓库内使用非常广泛
- 但定义并不一定集中在单一共享文件里，部分页面会内嵌自己的 `ToolTip` 实现
- 因此分析页面时，不能只看调用点，还要确认该页面实际使用的是哪一个 `ToolTip` 定义

### 使用判断

- 当普通页面需要错误弹窗时，优先检查当前页面是否已有 `ToolTip(...)` 可复用
- 如果是已有页面内广泛使用的交互，不要轻易另起一套弹窗实现
- 如果页面同时存在错误提示与下载进度提示，优先确认是否都由同一个 `ToolTip(...)` 能力承担
- 如果当前页面属于 Gen 流程，默认不要直接复用这里的错误弹窗模式，应先检查该 Gen 页面自己的错误处理方法
